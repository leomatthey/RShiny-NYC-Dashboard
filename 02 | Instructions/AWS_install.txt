Type of image Ubuntu Server 24.04 LTS SSD Volume type 64 bits t2.large 8Gb RAM
Create a new pair or keys: .pem, RSA. Create a name for the file and store in a secure place.
Network: Allow SSH, HTTPS and HTTP
Storage: 16Gb

Connect to instance: user ubuntu. Via AWS web (EC2 Instance Connect) or SSH (ssh -i "DAR_2025.pem" ubuntu@ec2-15-237-123-213.eu-west-3.compute.amazonaws.com) where DAR_2025.pem is the .pem file you have created in the setup process

sudo apt update
sudo apt upgrade
sudo apt install r-base-dev
sudo apt install libcurl4-openssl-dev libssl-dev libxml2-dev libfontconfig1-dev libharfbuzz-dev libfribidi-dev libfreetype6-dev libpng-dev libtiff5-dev libjpeg-dev
sudo R

In R: install.packages(c('tidyverse', 'shiny')). Takes a while because all packages need to be comppiled, be patient.


RStudio Server installation:
In the AWS instance console:
sudo apt-get install gdebi-core
(look for the command at https://posit.co/download/rstudio-server/ according to your Ubuntu version) wget https://download2.rstudio.org/server/jammy/amd64/rstudio-server-2026.01.1-403-amd64.deb
sudo gdebi rstudio-server-2026.01.1-403-amd64.deb
sudo rstudio-server start
Ensure that port 8787 is open in your security group settings on AWS
http://your-instance-ip:8787

Shiny Server installation:
In the AWS instance console:
(look for the command at https://posit.co/download/shiny-server/ according to your Ubuntu version) wget https://download3.rstudio.org/ubuntu-20.04/x86_64/shiny-server-1.5.23.1030-amd64.deb
sudo gdebi shiny-server-1.5.23.1030-amd64.deb
Ensure that port 3838 is open in your security group settings on AWS
http://your-instance-ip:3838

Managing ports:
Step 1: Navigate to the AWS EC2 instances
Step 2: Select Your Instance
Step 3: Select Security Tab -> Security Groups -> Inbound rules -> Edit inbound rules.
Step 4: Add rule -> Type: Custom TCP, Port Range: 8787, Source: Anywhere IPv4 , 0.0.0.0/0 (This allows access from any IP address. You might want to restrict it to specific IPs based on your needs.), Description (optional): RStudio Server
Step 5: Add rule -> Type: Custom TCP, Port Range: 3838, Source: Anywhere IPv4, 0.0.0.0/0 (This allows access from any IP address. You might want to restrict it to specific IPs based on your needs.), Description (optional): Shiny Server
Save changes

Test in your internet browser (Chrome, Firefox, etc). 
http://public_IPv4_adress:3838/ -> A Shiny server demo will appear
http://public_IPv4_adress:8787/ -> RStudio login page will appear
WARNING! Maybe at ESADE, using EDUROAM wifi, you cannot reach the machine.

To enter in RStudio Server: Create a new user in the EC2 instance. In the shell console:
sudo adduser new_user_name and follow the steps.

Note that you are using ubuntu user to connect to the machine and new_user_name to connect to Rstudio Server. Using EC2 Instance Connect to reach the machine you can change ubuntu to new_user_name if you want.
